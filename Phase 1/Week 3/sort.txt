📌 How Sorting Works in Elasticsearch
    Sorting in Elasticsearch is not based on _score unless you explicitly sort by _score.

    There are two completely different ranking systems:
    Mechanism	Purpose
    _score	    relevance ranking (BM25)
    sort	    deterministic ordering

    The moment you add "sort", Elasticsearch stops ranking by relevance (unless _score is explicitly included).

    Example:
        POST movies/_search
        {
          "query": {
            "match": {
              "title": "thriller"
            }
          },
          "sort": [
            { "release_year": "desc" }
          ]
        }
        In this example, the search results will be sorted by release_year in descending order, and the relevance score will not influence the order of the results.

        Fallback to relevance ranking:
            "sort": [
              { "release_year": "desc" },
              { "_score": "desc" }
            ]
            In this case, if there are multiple documents with the same release_year, they will be further sorted by their relevance score (_score) in descending order.


        How Elasticsearch Actually Sorts (Internally)
            Lucene uses doc_values.
            doc_values are a column-oriented structure stored on disk.
            Think of it like:
                doc_id	rating
                1	    7.2
                2	    9.1
                3	    5.5
            Sorting simply reads that column and orders docs.
            Important consequence:
                Sorting does NOT read _source.
                It uses doc_values only.
                This is why sorting is fast and efficient, even on large datasets.

        Sorting Arrays
        ==============
        Suppose a document has:
            "ratings": [9, 4, 7]
        Which value should be used for sorting?
        Elasticsearch needs a rule:
            mode	meaning
            =====================
            min	    smallest value
            max	    largest value
            avg	    average
            sum	    sum

        Example:
        "sort": [
          {
            "ratings": {
              "order": "desc",
              "mode": "max"
            }
          }
        ]
