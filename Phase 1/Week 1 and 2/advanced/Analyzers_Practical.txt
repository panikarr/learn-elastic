🧩 Analyzer 2 Goal — Log / Structured Text

    You need to design a custom analyzer for a field that stores log-like text.

        Example input documents
            ERROR[500]: /api/v1/users?id=123
            WARN auth token-expired user=77
            INFO PaymentService started

    🎯 What the analyzer should achieve

        When indexed, the text should be searchable by meaningful parts.

        So from: ERROR[500]: /api/v1/users?id=123
        you should be able to search and match by:

            error
            500
            api
            v1
            users
            id
            123

        Case-insensitive search is required.
        Accents don’t matter.

        We do NOT care about:
                grammar
                stemming
                stopwords
                synonyms

        We care about:
            extracting meaningful tokens from structured text

        🧠 Constraints
            Your analyzer should:
                Work well for logs / URLs / API paths
                Ignore punctuation and separators
                Be case-insensitive
                Keep numbers searchable
                Avoid over-processing


        Solution:

            Index design:
                PUT log_analyzer_test
                {
                  "settings": {
                    "analysis": {
                      "tokenizer": {
                        "logs_tokenizer":{
                          "type": "pattern",
                          "pattern": "[^a-zA-Z0-9]+"
                        }
                      },
                      "analyzer": {
                        "logs_analyzer":{
                          "tokenizer": "logs_tokenizer",
                          "filter":["lowercase"]
                        }
                      }
                    }
                  },
                  "mappings": {
                    "properties": {
                      "log_text": {
                        "type": "text",
                        "analyzer": "logs_analyzer"
                      }
                    }
                  }
                }

            Test:
                POST log_analyzer_test/_analyze
                {
                  "field": "log_text",
                  "text": "WARN auth token-expired user=77"
                }
            Ingest documents:
                PUT log_analyzer_test/_doc/1
                {
                  "log_text":"ERROR[500]: /api/v1/users?id=123"
                }

                PUT log_analyzer_test/_doc/2
                {
                  "log_text":"WARN auth token-expired user=77"
                }

                PUT log_analyzer_test/_doc/3
                {
                  "log_text":"INFO PaymentService started"
                }


            Search:
                POST log_analyzer_test/_search
                {
                  "query": {
                    "match": {
                      "log_text": "started"
                    }
                  }
                }
