
PUT test_mullti_fields
{
  "mappings": {
    "properties": {
      "title":{
        "type": "text",
        "fields": {
          "raw":{
            "type":"keyword"
          }
        }
      }
    }
  }
}

PUT test_mullti_fields/_doc/1
{
  "title": "Learn elastic search"
}

PUT test_mullti_fields/_doc/2
{
  "title": "Learn AI powered search"
}

POST test_mullti_fields/_search
{
  "query": {
    "match_all": {}
  }
}

//fails
POST test_mullti_fields/_search
{
  "query": {
    "term": {
      "title": {
        "value": "Learn elastic search"
      }
    }
  }
}

//works
POST test_mullti_fields/_search
{
  "query": {
    "term": {
      "title.raw": {
        "value": "Learn elastic search"
      }
    }
  }
}

//works
POST test_mullti_fields/_search
{
  "query": {
    "bool": {
      "filter": [
        {
          "term": {
            "title.raw": "Learn elastic search"
          }
        }
      ]
    }
  }
}


//works
POST test_mullti_fields/_search
{
  "query": {
    "match_phrase": {
      "title": "Learn elastic search"
    }
  }
}

match_phrase works by using positional information stored in the inverted index.
Lucene records the position of each token within a document.
When a phrase query runs, Elasticsearch checks whether the tokens appear in the correct order and distance using these positions.
It does not build phrases in memory, so heap usage is minimal; the operation mostly uses disk-backed index data and CPU for positional checks.
